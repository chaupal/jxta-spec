Goals

This project's responsibility is to ensure the standardization of the JXTA Core Protocols and ensure that the various implementations of those core protocols are compliant. This means:

  1. Developing a charter for the Project. This will describe what our goals are for the Project for the next 3 months, 6 months, 1 year, 2 years.
  2. the protocol. This will be the basis for discussion in the community for determining what should be in scope and out of scope for the core protocols. The starting point is the existing description of the JXTA protocols.
  3. Managing the input from the JXTA/IETF/IRTF community in the development of the protocols.
  4. Documenting the protocol and changes as they are discussed and implemented.
  5. Developing normative XML Schemas and/or DTDs for the protocols.
  6. Developing test suites
  7. Perhaps even monitoring tools
  8. Review breaking the specification into multiple documents and maintaining a bill of documents that refer to the individually controlled specifications.

Status

The JXTA Specification was submitted to the IETF June of 2002 in anticpation of forming a working group. The IETF declined to start a working group for JXTA, but instead referred JXTA to their sister organization IRTF. JXTA is expected to be a key part of the IRTF's Peer-to-Peer working group. See JXTA IETF Status Page for more information.

Currently, specification work and translation of the existing specification is ongoing. Project owners encourage everybody in the community to participate in this project and help define and specify the JXTA protocols.

The specifications linked to from this page are update on a rougly quarterly/bi-yearly basis to accomodate IETF schedules. For the latest specification updates, consult the CVS repository directly (source).
